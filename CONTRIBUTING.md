# Contributing to Sweep Bros

Thank you for your interest in contributing to Sweep Bros! We welcome contributions from the community to help make this project even better.

## Getting Started

1.  **Fork the repository**: Click the "Fork" button at the top right of this page.
2.  **Clone your fork**:
    ```bash
    git clone https://github.com/YOUR-USERNAME/sweep-bros.git
    cd sweep-bros
    ```
3.  **Run locally**:
    - Simply open `index.html` in a supported browser (Chrome, Edge, Opera, Brave).
    - Alternatively, use a local server (e.g., Live Server in VS Code) for a better development experience.

## Development Workflow

1.  Create a new branch for your feature or fix:
    ```bash
    git checkout -b feature/your-feature-name
    ```
2.  Make your changes.
3.  Test your changes manually in the browser.

## Project Structure

- `index.html`: The main entry point.
- `css/`: Stylesheets (Vanilla CSS).
- `js/`: JavaScript source code (ES Modules).
  - `js/app.js`: Entry point script.
  - `js/core/`: Core application logic classes.
  - `js/modules/`: Feature-specific modules.

## Pull Request Guidelines

1.  Ensure your code follows the existing style and structure.
2.  Provide a clear description of your changes in the Pull Request.
3.  If adding a new feature, please update the README if necessary.
